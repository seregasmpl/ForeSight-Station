# Форсайт-Станция — Дизайн-документ

## Обзор

**Форсайт-Станция** — веб-приложение для школьного форсайт-мероприятия "Форсайт космоса" (школа "Сколка", 10 апреля). 40-50 старшеклассников в 8 командах работают с ИИ-агентами (технооптимист / технопессимист), которые помогают готовить аргументы к дебатам о будущем космоса к 2100 году. Агенты опираются на коллекции научно-фантастических и научно-популярных текстов.

## Структура мероприятия

- 4 темы: "Человечество — космическая цивилизация", "Неисчерпаемые блага космоса", "Тайны космоса в наших руках", "Земля — наш космический корабль"
- 8 команд: по 2 на тему (оптимист + пессимист)
- Каждая команда общается с ОДНИМ агентом (своей роли)
- Тема команды фиксирована на всю сессию
- Дебаты происходят вживую между командами, ИИ — консультант

## Архитектура

### Стек

| Компонент | Технология |
|-----------|-----------|
| Backend | Python 3.12, FastAPI, uvicorn |
| Frontend | Vanilla HTML/CSS/JS (статика, раздаётся FastAPI) |
| BM25 поиск | SQLite FTS5 + PyStemmer (Snowball Russian) |
| Dense поиск | `intfloat/multilingual-e5-small` (118MB, CPU) |
| LLM | OpenRouter API (Claude / GPT-4 / Gemini) |
| Хранение | Всё в памяти (dict), без внешних БД |

### Коллекции текстов

- **2 коллекции**: одна для оптимиста (утопическая фантастика, вдохновляющий научпоп), одна для пессимиста (предупреждающая фантастика, критические материалы)
- Тексты раскладываются в `data/collections/optimist/` и `data/collections/pessimist/` как `.txt` файлы
- При старте сервера тексты нарезаются на чанки, индексируются (BM25 + эмбеддинги)
- Индексация ~50-60 текстов на CPU: 5-10 минут (один раз)

## Пайплайн запроса

```
Команда вводит вопрос
        ↓
   [POST /api/ask]
        ↓
   Определение агента по сессии (оптимист или пессимист)
        ↓
   Гибридный поиск по коллекции агента:
     ├── BM25 (SQLite FTS5, русский стеммер) → top-50
     ├── Dense (multilingual-e5-small, cosine) → top-50
     └── RRF (Reciprocal Rank Fusion, k=60) → merged
        ↓
   MMR (Maximal Marginal Relevance) → top-10 разнообразных чанков
        ↓
   Дедупликация: штраф -0.3...-0.5 для чанков, использованных ранее в сессии
        ↓
   Сборка промпта:
     ├── Системный промпт (роль + правила)
     ├── Тема сессии
     ├── 10 чанков из коллекции
     ├── Фокус-линза (случайный акцент)
     └── Краткая история предыдущих вопросов сессии
        ↓
   OpenRouter LLM вызов (async)
        ↓
   Парсинг ответа → 7 секций
        ↓
   Сохранение в историю сессии + обновление использованных чанков
        ↓
   JSON ответ на фронт
```

## Агенты

### Общая структура промпта

```
[1] РОЛЬ — технооптимист или технопессимист
[2] ТЕМА — одна из 4 тем
[3] КОНТЕКСТ ИЗ ТЕКСТОВ — 10 чанков из RAG (после MMR + дедуп)
[4] ФОКУС-ЛИНЗА — случайный акцент для разнообразия
[5] ИСТОРИЯ — предыдущие вопросы команды (краткий пересказ)
[6] ФОРМАТ ОТВЕТА — 7 секций (строго)
```

### Персоны

**Технооптимист:**
- Верит в прогресс, признаёт риски, но видит решения
- Тон: вдохновляющий, аргументированный
- Установка: "Каждое технологическое ограничение — инженерная задача, которая будет решена"

**Технопессимист:**
- Видит возможности, но фокусируется на цене, рисках, непредвиденных последствиях
- Тон: критичный, провокационный, конструктивный
- Установка: "Вопрос не 'можем ли мы', а 'должны ли мы' и 'какой ценой'"

### Работа с фантастикой

Агенты НЕ "играют роль" конкретных авторов. Они ссылаются на произведения из своей коллекции как на источник образов и идей:

> "В твоей библиотеке — коллекция научно-фантастических и научно-популярных текстов. Ссылайся на конкретные сцены и идеи из них для иллюстрации своих аргументов. Формат: «В произведении "[название]" описана ситуация, когда...». Фантастика — инструмент мышления, не доказательство."

### Формат ответа (7 секций)

1. **Суть позиции** — 2-3 предложения, тезис
2. **Ключевые аргументы** — 5 пунктов, с цитатами из фантастики
3. **Что произойдёт к 2100** — 4-5 конкретных прогнозов
4. **Главные риски / ограничения** — 4-5 пунктов
5. **Что сказать на дебатах** — готовая реплика, 3-4 предложения
6. **Вопросы оппонентам** — 2 острых вопроса
7. **Новость из 2100 года** — яркая выдуманная новость (1 абзац)

## Механизмы разнообразия ответов

### 1. MMR (Maximal Marginal Relevance)
При выборе чанков штрафует за похожесть с уже отобранными. Вместо "10 самых релевантных" — "10 релевантных И разнообразных". Параметр lambda=0.7 (баланс релевантность/разнообразие).

### 2. Чанк-дедупликация по сессии
В памяти сессии хранится множество ID чанков, использованных в предыдущих ответах. При следующем запросе их RRF-скор снижается на 0.3-0.5. Не исключение, а штраф — если чанк критически важен, он всё равно попадёт.

### 3. Фокус-рулетка
При каждом запросе в промпт добавляется случайный фокус из списка:
- социальные последствия
- технологические прорывы
- этические дилеммы
- повседневная жизнь людей
- геополитические сдвиги
- неожиданные побочные эффекты
- культурные трансформации
- экологические аспекты

## Сессии

### Создание (учитель)
Учитель через админку создаёт 8 сессий, для каждой выбирая тему и роль. Система генерирует код (например: ЛУНА-01, ОРБИТА-02).

### Вход (команда)
- Команда вводит код на главной странице
- Если код валидный и сессия свободна — вход, сессия блокируется
- Если сессия уже занята — отказ: "Этот экипаж уже на борту"
- Heartbeat (каждые 30 секунд) — если нет пингов 2 минуты, сессия разблокируется

### Данные сессии (в памяти)
```python
{
    "code": "ЛУНА-01",
    "topic": "Человечество — космическая цивилизация",
    "role": "optimist",
    "status": "active" | "waiting" | "disconnected",
    "connected_at": datetime | None,
    "questions": [
        {"question": "...", "answer": {...}, "chunks_used": [...], "timestamp": ...}
    ],
    "used_chunk_ids": set()
}
```

### Отображение кода
Код сессии крупно отображается в углу экрана команды — учитель может ходить по классу и видеть кто где сидит.

## Админка

### Вход
`/admin` — простой пин-код (задаётся в config.py). Без сложной авторизации.

### Дашборд "Центр управления полётами"
- Таблица всех 8 сессий: код, тема, роль, статус, количество вопросов
- Клик на сессию — просмотр истории вопросов и ответов команды
- Кнопка создания/пересоздания сессий
- Статусы: ожидает (серый), онлайн (зелёный), отключена (красный)

## Фронтенд — советский ретрофутуризм

### Визуальный стиль
- **Фон**: тёмно-синий/чёрный космос, анимированные звёзды (CSS)
- **Панели**: скруглённые "иллюминаторы", металлическая рамка
- **Шрифт**: моноширинный (`Space Mono` или аналог)
- **Цвета оптимиста**: янтарный/золотой текст на тёмном фоне
- **Цвета пессимиста**: зелёный/голубой текст на тёмном фоне
- **Ввод**: стилизован как командная строка `> ЗАПРОС ЭКИПАЖА: _`
- **Загрузка**: "БОРТОВОЙ КОМПЬЮТЕР ОБРАБАТЫВАЕТ ЗАПРОС..." с мигающим курсором
- **Код сессии**: крупно в углу, стилизован как позывной

### Страницы
1. **index.html** — ввод кода сессии (одно поле, космический фон)
2. **session.html** — рабочий экран команды (чат с агентом, история, код в углу)
3. **admin.html** — дашборд учителя (таблица команд, детали сессий)

### Атмосфера
- Заголовок: "ФОРСАЙТ-СТАНЦИЯ" или "БОРТОВОЙ ПРОГНОСТИЧЕСКИЙ КОМПЛЕКС"
- Темы как миссии: "МИССИЯ 1: ЧЕЛОВЕЧЕСТВО — КОСМИЧЕСКАЯ ЦИВИЛИЗАЦИЯ"
- Анимация при получении ответа (передача данных)
- Звуковые эффекты опциональны

## Ограничения и допущения

- Максимум 8 одновременных сессий
- Все данные в памяти — перезапуск сервера обнуляет сессии
- CPU only — без GPU
- Тексты на русском языке
- Индексация при старте (5-10 минут)
- Один сервер, локальная сеть или интернет
