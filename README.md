# Форсайт-Станция

Веб-приложение для школьной форсайт-сессии о будущем космоса к 2100 году. 8 команд (4 темы x 2 роли: технооптимист / технопессимист) готовят аргументы для дебатов с помощью ИИ-консультантов.

## Архитектура

- **Backend**: FastAPI + Uvicorn
- **RAG**: гибридный поиск (BM25 через SQLite FTS5 + dense embeddings через `intfloat/multilingual-e5-small`) с RRF-слиянием и MMR-отбором
- **LLM**: OpenRouter API (по умолчанию `google/gemini-3-flash-preview`)
- **Frontend**: ванильный HTML/CSS/JS в космической стилистике

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. API-ключ

Создайте файл `.env` в корне проекта:

```
OPENROUTER_API_KEY=sk-or-v1-ваш-ключ
```

### 3. Текстовые коллекции

Положите `.txt` файлы в:
- `data/collections/optimist/` — тексты для технооптимистов
- `data/collections/pessimist/` — тексты для технопессимистов

В репозитории уже есть подборка произведений из Public Domain (Циолковский, Толстой, Верн, Уэллс, Замятин, Богданов, Одоевский).

### 4. Запуск

```bash
python serve.py
```

Сервер стартует на `http://localhost:8001`. При запуске автоматически индексирует текстовые коллекции.

## Как играть

1. Учитель открывает **Центр управления полётами** (`/admin`) и вводит PIN (по умолчанию `1234`)
2. Нажимает **Создать все сессии** — генерируются 8 кодов (ЛУНА-01, ЛУНА-02, ОРБИТА-01, ...)
3. Каждая команда вводит свой код на главной странице и попадает в чат
4. Команды задают вопросы ИИ-консультанту, который помогает готовить аргументы для дебатов
5. Каждый консультант отвечает по 7 разделам: позиция, аргументы, прогнозы, риски, речь для дебатов, вопросы оппонентам, новость из 2100 года

## Структура проекта

```
serve.py              — FastAPI-сервер, эндпоинты API
agents.py             — взаимодействие с LLM (OpenRouter)
sessions.py           — управление сессиями, персистентность (JSON)
indexer.py            — индексация текстов (FTS5 + dense embeddings)
retriever.py          — гибридный RAG-поиск (BM25 + dense + RRF + MMR)
models.py             — Pydantic-модели данных
config.py             — конфигурация (загрузка .env)
prompts/              — системные промты для оптимиста и пессимиста
static/               — фронтенд (HTML, CSS, JS)
data/collections/     — текстовые коллекции для RAG
tests/                — тесты (pytest + Playwright)
```

## Конкурентность

Приложение поддерживает одновременную работу всех 8 команд:
- `asyncio.Lock` на RAG-retriever (модель эмбеддингов не потокобезопасна)
- `asyncio.Lock` на мутации сессий
- Атомарная запись `sessions.json` (через `os.replace`)
- LLM-запросы выполняются параллельно через async HTTP

## Конфигурация

Основные параметры в `config.py`:

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `OPENROUTER_MODEL` | `google/gemini-3-flash-preview` | Модель LLM |
| `LLM_TEMPERATURE` | `0.8` | Температура генерации |
| `LLM_TIMEOUT` | `120.0` | Таймаут LLM-запроса (сек) |
| `ADMIN_PIN` | `1234` | PIN для админ-панели |
| `CHUNK_SIZE` | `1000` | Размер чанка (символов) |
| `MMR_TOP_K` | `10` | Количество чанков для контекста |
